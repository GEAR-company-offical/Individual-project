# Отчёт  
## Размещение заготовки персонального сайта на GitHub Pages

---

## 1. Установка необходимого программного обеспечения

Для работы с репозиторием и размещения сайта были установлены следующие инструменты:

- Git — система контроля версий.
- Visual Studio Code (или другой редактор кода) — для редактирования файлов сайта.
- Веб-браузер — для проверки отображения сайта.

Проверка установки Git:

```bash
git --version
```

## 2. Скачивание шаблона темы сайта

Для создания персонального сайта был выбран готовый шаблон темы.

Шаблон был скачан одним из способов:

### Вариант 1 — через Git

```bash
git clone https://github.com/username/theme-name.git
```

### Вариант 2 — через ZIP-архив

- Скачать архив с GitHub
- Распаковать его в рабочую папку

После скачивания была проверена структура проекта.


![alt text](src/image02.png)
---

## 3. Размещение шаблона на хостинге GitHub

1. Создан новый репозиторий формата:

```
username.github.io
```


![alt text](src/image03.png)

2. В папке проекта выполнена инициализация Git:

```bash
git init
git add .
git commit -m "Initial commit"
```


![alt text](src/i0image.png)
3. Подключён удалённый репозиторий:

```bash
git remote add origin https://github.com/username/username.github.io.git
```

4. Отправка проекта на GitHub:

```bash
git push -u origin master
```


![alt text](src/image-1.png)
---

## 4. Установка параметра URL сайта

В конфигурационном файле проекта (например, `_config.yml` для Jekyll) был установлен параметр:

```yml
url: "https://username.github.io"
baseurl: ""
```

Это необходимо для корректной работы внутренних ссылок сайта.

---

## 5. Размещение сайта на GitHub Pages


![alt text](src/image-2.png)


1. В настройках репозитория:
   - Перейти в Settings → Pages
   - Выбрать ветку master
   - Сохранить изменения

2. После автоматической сборки сайт стал доступен по адресу:

```
https://username.github.io
```

3. Корректность отображения сайта проверена в браузере.


![alt text](src/image-3.png)

В ходе выполнения работы была размещена заготовка персонального сайта на GitHub Pages.

Были выполнены следующие этапы:

- Установлено необходимое программное обеспечение
- Скачан шаблон темы
